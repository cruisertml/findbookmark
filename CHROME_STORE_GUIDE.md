# Chrome Web Store 上架指南

本指南将帮助您将书签搜索工具扩展上架到 Chrome Web Store。

## 📋 上架前准备

### 1. 准备扩展图标

扩展需要以下尺寸的图标：
- **16x16** - 扩展程序页面和上下文菜单
- **48x48** - 扩展管理页面
- **128x128** - Chrome Web Store 和安装对话框

**操作步骤：**
1. 打开 `create-icons.html` 文件
2. 点击各尺寸的"下载"按钮
3. 将下载的文件保存到 `icons` 目录

### 2. 准备商店截图和宣传图

Chrome Web Store 需要以下图片：

#### 必需图片：
- **至少1张截图**（1280x800 或 640x400 像素）
  - 展示扩展的主要功能
  - 建议：展示搜索界面、编辑功能等

#### 可选但推荐：
- **小宣传图**（440x280 像素）- 用于商店列表页
- **大宣传图**（920x680 像素）- 用于商店详情页

**建议截图内容：**
1. 搜索书签的界面
2. 编辑书签的功能
3. 搜索结果展示

### 3. 准备详细描述

需要准备以下文本内容：

#### 简短描述（132字符以内）
```
快速搜索所有书签，支持模糊匹配，并可编辑和删除书签。让书签管理更简单高效。
```

#### 详细描述（可以包含功能列表）
```
书签搜索工具是一个强大的Chrome扩展，帮助您快速管理和查找书签。

主要功能：
• 🔍 快速搜索：支持模糊匹配，实时搜索所有书签
• ✏️ 编辑书签：轻松修改书签名称
• 🗑️ 删除书签：一键删除不需要的书签
• ⌨️ 快捷键支持：编辑时支持Enter保存、ESC取消

使用方法：
1. 点击扩展图标打开搜索界面
2. 在搜索框中输入关键词
3. 查看匹配的书签结果
4. 点击"编辑"修改书签名称
5. 点击"删除"移除不需要的书签

让您的书签管理更简单、更高效！
```

### 4. 选择分类和标签

- **分类**：生产力工具 (Productivity)
- **标签**：书签、搜索、管理、工具

### 5. 隐私政策（如果需要）

如果扩展收集用户数据，需要提供隐私政策URL。本扩展只访问书签数据，不收集任何信息，但建议创建一个简单的隐私政策页面。

## 🚀 上架步骤

### 第一步：创建开发者账号

1. 访问 [Chrome Web Store 开发者控制台](https://chrome.google.com/webstore/devconsole)
2. 使用 Google 账号登录
3. 支付一次性注册费用：**$5 USD**（约35元人民币）
   - 支持信用卡、借记卡或 PayPal
   - 这是一次性费用，永久有效

### 第二步：打包扩展

1. **准备打包文件**
   - 确保所有文件都在项目根目录
   - 确保 `icons` 目录下有所有必需的图标文件

2. **创建 ZIP 文件**
   - Windows: 选中所有文件 → 右键 → 发送到 → 压缩(zipped)文件夹
   - Mac: 选中所有文件 → 右键 → 压缩
   - 确保 ZIP 文件包含：
     - `manifest.json`
     - `popup.html`
     - `popup.js`
     - `popup.css`
     - `icons/` 目录（包含所有图标）

3. **检查清单**
   - ✅ manifest.json 格式正确
   - ✅ 所有图标文件存在
   - ✅ 没有测试代码或调试信息
   - ✅ ZIP 文件大小不超过 10MB

### 第三步：提交扩展

1. **进入开发者控制台**
   - 访问 https://chrome.google.com/webstore/devconsole
   - 点击"新增项目"

2. **上传 ZIP 文件**
   - 点击"选择文件"上传打包好的 ZIP
   - 等待上传完成

3. **填写商店信息**

   **基本信息：**
   - **名称**：书签搜索工具（或您喜欢的名称）
   - **简短描述**：快速搜索所有书签，支持模糊匹配，并可编辑和删除书签
   - **详细描述**：使用上面准备的详细描述
   - **类别**：生产力工具
   - **语言**：中文（简体）

   **图片：**
   - 上传至少1张截图（1280x800 或 640x400）
   - 可选：上传宣传图

   **隐私设置：**
   - 如果扩展不收集数据，选择"不收集用户数据"
   - 如果需要，提供隐私政策URL

4. **定价和分发**
   - **定价**：免费
   - **可见性**：
     - 公开：所有人可见（推荐）
     - 不公开：仅通过链接访问
     - 信任的测试人员：仅限测试

5. **提交审核**
   - 检查所有信息无误
   - 点击"提交审核"
   - 等待审核（通常1-3个工作日）

## 📝 审核注意事项

### 常见拒绝原因：

1. **权限说明不清晰**
   - ✅ 已解决：manifest.json 中只请求了 `bookmarks` 权限，这是必需的

2. **功能描述不准确**
   - ✅ 确保描述与实际功能一致

3. **截图质量差**
   - ✅ 使用清晰的截图，展示实际功能

4. **隐私政策缺失**（如果收集数据）
   - ✅ 本扩展不收集数据，选择"不收集用户数据"即可

5. **代码质量问题**
   - ✅ 确保代码没有明显错误
   - ✅ 移除调试代码

## 🔄 更新扩展

上架后如需更新：

1. 修改代码
2. 更新 `manifest.json` 中的版本号
3. 重新打包 ZIP
4. 在开发者控制台选择扩展 → 上传新版本
5. 提交审核

## 📊 版本号规范

在 `manifest.json` 中使用语义化版本号：
- 格式：`主版本号.次版本号.修订号`
- 示例：`1.0.0` → `1.0.1`（小修复）→ `1.1.0`（新功能）→ `2.0.0`（重大更新）
